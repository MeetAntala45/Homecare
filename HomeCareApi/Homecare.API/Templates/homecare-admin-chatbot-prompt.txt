=== ADMIN LOGIN ===
Admin logs in using email and password.
Password must be 8 to 15 characters and  include alphanumeric and special characters.
Email and password are required fields with valid email format check.
Password field has a show/hide eye icon.

=== FORGOT PASSWORD ===
Admin clicks Forgot Password on the login page and enters their registered email.
A password reset link is sent to the registered email.
The reset link expires in 10 minutes. After expiry it cannot be used.
On clicking the reset link, a reset password page opens with email shown as disabled.
Admin enters New Password and Confirm Password — both must match.
On success, admin is redirected to login page with a success message.

=== ADMIN PROFILE ===
Admin can view and manage their profile from the My Profile page.
Profile shows: profile photo, user name, role, mobile number, email address, address, and password placeholder.
Admin can upload or change profile photo by hovering over it and clicking the edit icon.
Admin can edit contact information: mobile number, email, and address via a modal.
Admin can change password via Change Password modal: requires current password, new password, and confirm password.
New password and confirm password must match. Current password must be correct.
All fields are required.

=== SERVICE TYPE MANAGEMENT (MASTER DATA) ===
Admin can view, add, edit, and delete service types from the Master Data page.
Listing shows Name (image + name) and Action columns with Manage, Edit, and Delete icons.
Add or Edit opens a popup with an image uploader and service type name textbox.
Service type name is a required field and must be unique in the database.
Duplicate names show an appropriate validation message.
Only image files are supported for upload with defined size limits.
Image upload is not mandatory. If no image, a default general image is shown.
Delete shows a confirmation popup: Are you sure you want to delete {Service Type Name}?

Sub Category section shows subcategories for the selected category with Add and Delete options.
Category and subcategory names are required fields with max length validation.
Duplicate category or subcategory names show an appropriate validation message.
Delete shows a confirmation prompt before removing.
